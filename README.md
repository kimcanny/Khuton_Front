# 🚀 Policy OS: 프론트엔드 프로젝트 안내서

안녕하세요! **Policy OS** 프론트엔드 프로젝트에 오신 것을 진심으로 환영합니다. 이 문서는 코딩이 처음이거나, 프론트엔드 개발 환경이 낯선 초보 기여자분들을 위해 작성되었습니다. 천천히 읽어보시면 누구나 프로젝트에 참여하실 수 있습니다.

---

# 📝 프로젝트 소개

이 프로젝트는 **확장 가능한 대시보드 스타일의 UI**를 구축하기 위한 프론트엔드 저장소입니다. 복잡한 정책 분석 데이터나 인포그래픽을 사용자에게 깔끔하고 세련되게 보여주는 것을 목표로 합니다.

*   **현대적인 도구**: React와 Vite를 사용하여 매우 빠르고 효율적으로 동작합니다.
*   **세련된 디자인**: Tailwind CSS를 활용하여 Apple 스타일의 미니멀하고 고급스러운 UI를 지향합니다.
*   **체계적인 구조**: **Atomic Design(아토믹 디자인)** 아키텍처를 도입하여, 작은 부품들을 조립해 큰 화면을 만드는 방식으로 개발합니다.

---

# 🛠 기술 스택

우리 프로젝트에서 사용하는 주요 도구들입니다.

*   **React**: 화면의 각 부분(컴포넌트)을 독립적으로 만들고 관리하게 해주는 라이브러리입니다.
*   **Vite**: 코드를 작성하고 브라우저에서 바로 확인할 수 있게 도와주는 아주 빠른 개발 도구입니다.
*   **Tailwind CSS**: 복잡한 CSS 파일 대신, HTML 태그 안에 미리 정의된 클래스 이름을 적어 스타일을 입히는 방식입니다.
*   **React Router**: 웹사이트 안에서 여러 페이지(메인, 리포트, 분석 등)를 이동할 수 있게 해줍니다.
*   **Git / GitHub**: 코드의 변경 이력을 기록하고, 여러 사람이 함께 협업할 수 있게 도와주는 도구입니다.
*   **npm**: 필요한 라이브러리(패키지)를 설치하고 관리하는 도구입니다.

# 💻 개발 환경 설치

프로젝트를 시작하기 전에 컴퓨터에 다음 도구들을 설치해야 합니다.

1.  **Node.js (LTS 버전)**: 자바스크립트 코드를 컴퓨터에서 실행하게 해주는 엔진입니다. [공식 사이트](https://nodejs.org/)에서 다운로드하세요.
2.  **npm (Node Package Manager)**: Node.js를 설치하면 자동으로 함께 설치됩니다.
3.  **Git**: 코드의 버전을 관리하기 위해 꼭 필요합니다. [Git 설치 가이드](https://git-scm.com/downloads)를 참고하세요.
4.  **VS Code (Visual Studio Code)**: 코드를 작성하는 가장 인기 있는 편집기입니다. 다양한 확장 기능을 사용할 수 있어 매우 편리합니다.

# 🚀 프로젝트 실행 방법

컴퓨터에 도구들이 설치되었다면, 아래 순서대로 실행해 보세요.

### 1. 저장소 복제 (Clone)
먼저 온라인에 있는 코드를 내 컴퓨터로 가져옵니다. 터미널(또는 CMD)을 열고 아래 명령어를 입력하세요.

```bash
git clone https://github.com/kimcanny/Khuton_Front.git
```

### 2. 폴더로 이동
가져온 프로젝트 폴더 안으로 들어갑니다.

```bash
cd Khuton_Front
```

### 3. 패키지 설치
프로젝트 실행에 필요한 라이브러리들을 한꺼번에 설치합니다.

```bash
npm install
```
*이 명령어는 `package.json`에 기록된 모든 도구들을 `node_modules`라는 폴더에 다운로드합니다.*

### 4. 개발 서버 실행
이제 실제 화면을 확인해 볼 차례입니다.

```bash
npm run dev
```
명령어를 입력하면 `localhost:5173` 같은 주소가 나옵니다. 이 주소를 브라우저에 입력하면 프로젝트 화면이 나타납니다!

# 🌿 브랜치 생성 및 협업 방법

협업할 때는 `main`이나 `dev` 브랜치에 직접 코드를 올리지 않는 것이 원칙입니다. 대신 나만의 **작업용 가지(Branch)**를 만들어야 합니다.

1.  **공용 개발 브랜치(`dev`)로 이동**:
    ```bash
    git checkout dev
    ```
2.  **내 작업 브랜치 만들기**:
    ```bash
    git checkout -b feature/내이름
    ```
    *예: `git checkout -b feature/hong-gildong`*

이렇게 하면 안전하게 나만의 공간에서 코드를 수정하고 나중에 합칠 수 있습니다.

# 📂 프로젝트 구조 설명

각 폴더는 저마다의 역할이 있습니다.

*   `src/`: 실제 우리가 작성하는 모든 코드가 담긴 곳입니다.
    *   `components/`: 화면을 구성하는 조각들이 모여 있습니다.
        *   `atoms/`: 가장 작은 단위 (버튼, 제목, 선 등).
        *   `molecules/`: 작은 조각들을 합친 단위 (아이템 하나, 카드 하나 등).
        *   `organisms/`: 더 큰 덩어리 (통계 섹션, 리포트 패널 등).
    *   `pages/`: 실제 주소마다 보여줄 전체 화면입니다 (Dashboard, Reports 등).
    *   `layouts/`: 여러 페이지에서 공통으로 사용하는 틀입니다 (상단바, 하단바 등).
    *   `router/`: 어떤 주소로 접속했을 때 어떤 페이지를 보여줄지 결정합니다.
    *   `hooks/`: 반복되는 로직을 재사용하기 위해 따로 빼놓은 곳입니다.
    *   `utils/`: 간단한 계산이나 날짜 형식 변경 같은 유틸리티 함수들이 있습니다.
    *   `styles/`: 전역 스타일 설정이 담겨 있습니다.

# 🎨 Atomic Design 설명

우리는 화면을 거대한 덩어리로 생각하지 않고, **레고 블록**처럼 생각합니다.

1.  **Atoms (원자)**: 더 이상 쪼갤 수 없는 기본 요소입니다.
    *   *예: 제목(`Title`), 설명글(`Subtitle`), 얇은 선(`Divider`)*
2.  **Molecules (분자)**: 원자들이 모여 하나의 기능을 합니다.
    *   *예: 제목과 숫자가 합쳐진 카드(`StatCard`), 상태 표시가 붙은 목록 아이템(`PolicyItem`)*
3.  **Organisms (유기체)**: 분자들이 모여 독립적인 섹션을 이룹니다.
    *   *예: 여러 카드가 모인 통계 구역(`StatsSection`), 전체 정책 목록 창(`PolicyPanel`)*
4.  **Pages (페이지)**: 최종적으로 사용자가 보게 되는 전체 화면입니다.

# 🔄 기본 개발 흐름

코드를 수정하고 공유하는 표준 과정입니다.

1.  **최신 코드 가져오기**: 작업을 시작하기 전 항상 `git pull origin dev`를 하세요.
2.  **브랜치 만들기**: `feature/기능이름` 브랜치를 만듭니다.
3.  **코드 작성**: VS Code에서 멋진 컴포넌트를 만듭니다.
4.  **로컬 테스트**: 브라우저(`localhost`)에서 잘 나오는지 확인합니다.
5.  **커밋 (Commit)**: 내가 한 일을 기록합니다. `git add .` -> `git commit -m "feat: 제목 추가"`
6.  **푸시 (Push)**: 내 브랜치를 GitHub에 올립니다. `git push origin 내브랜치이름`
7.  **PR 생성**: GitHub 웹사이트에서 `Pull Request`를 만들어 팀원들의 검토를 받습니다.

# 📏 Git 사용 규칙

*   **`node_modules`는 절대 올리지 마세요**: `.gitignore` 파일이 이미 이 폴더를 걸러주고 있으니 걱정 마세요.
*   **자주 상태 확인**: `git status`를 수시로 입력해서 어떤 파일이 수정되었는지 확인하세요.
*   **의미 있는 커밋 메시지**: 내가 무엇을 했는지 짧고 명확하게 적어주세요.
    *   `feat: `: 새로운 기능 추가
    *   `fix: `: 버그 수정
    *   `docs: `: 문서 수정 (README 등)
    *   `style: `: 디자인/스타일 변경

# ❓ 트러블슈팅 (문제 해결)

**1. `npm install`이 실패해요!**
*   인터넷 연결을 확인하고, Node.js 버전이 너무 낮지는 않은지 확인해 보세요. 가끔 `node_modules` 폴더를 지우고 다시 실행하면 해결되기도 합니다.

**2. 화면이 안 나와요 (`npm run dev` 오류)**
*   터미널에 빨간색 에러 메시지가 있는지 보세요. 대부분 파일 경로를 잘못 적었거나, 괄호를 닫지 않았을 때 발생합니다.

**3. 주소가 5173이 아니라 5174로 나와요!**
*   이미 5173 포트를 다른 프로그램이 사용 중일 때 자동으로 숫자가 올라갑니다. 당황하지 말고 터미널에 나온 주소 그대로 접속하시면 됩니다.

**4. 디자인이 적용되지 않아요 (Tailwind 문제)**
*   `className`에 오타가 없는지 확인해 보세요. 또한 `index.css` 상단에 `@import "tailwindcss";`가 잘 있는지 확인해 보세요.

# 🤝 협업 시 주의사항

*   **App.jsx를 너무 무겁게 만들지 마세요**: 모든 코드를 한 파일에 다 넣으면 나중에 수정하기 매우 힘듭니다. 기능을 나누어 `components/` 폴더에 분리하세요.
*   **재사용성을 생각하세요**: 한 번 만든 버튼은 다른 곳에서도 쓸 수 있도록 범용적으로 만드세요.
*   **이름은 명확하게**: `Item1`, `List2` 같은 이름보다는 `StatCard`, `PolicyList`처럼 역할을 바로 알 수 있는 이름을 사용하세요.
